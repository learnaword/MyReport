# 数据库设计：毕业生就业去向宽表

**需求来源：** 用户提供现行导入表头（56 列）；此前含代码列/500 强等宽表头已收缩  
**功能短名：** `graduate_employment`  
**目标库：** MySQL 8（与本仓库 `application.yml` / JPA `ddl-auto=update` 对齐）  
**设计原则：** 以「一人一行」分析宽表为主，字段与导入表头一一对应；库内可保留历史扩展列（新表头不再导入时置空）。

---

## 1. 范围与实体

| 实体 | 表名 | 说明 |
|------|------|------|
| 毕业生就业去向记录 | `grad_employment_record` | 主事实表；对应导入/分析的一行毕业生数据 |
| 学校 | `school` | 「学校名称」导入时精确匹配 `school.name` → `school_id` |

本期主表：**`grad_employment_record`**；学校由独立 `school` 表维护。

## 4. 约束与索引策略

| 项 | 结论 | 说明 |
|----|------|------|
| 代理主键 | `id` | 避免学号变更/重复导入破坏引用 |
| 业务唯一 | `(school_id, student_no, graduation_year)` | 同校同届同学号再导入覆盖更新 |
| 学号非空 | 建议业务层校验 | 导入脏数据时允许暂空再清洗；落库唯一键要求两者均非空 |
| 字符集 | `utf8mb4` | 覆盖生僻字姓名、单位名 |
| 统计索引 | 见 DDL | 按年、学院专业、去向、就业省市 |

## 6. 已确认结论

| # | 议题 | 结论 |
|---|------|------|
| 1 | 三次「单位名称」 | **不合并**，保留 `employer_name` / `unit_name_after_occupation` / `further_study_school_name` |
| 2 | 业务唯一键 | **学校ID + 学号 + 毕业年份** 唯一；同键再导入覆盖更新 |
| 3 | 学校字段 | 现行表头用 **学校名称** 匹配 `school` 表；`school_id` 落库 |
| 4 | 就业地理双口径 | **并存**：`job_econ8`/`job_econ8_2`，`job_in_out_province`/`job_in_out_province_2` |
| 5 | QS / US 排名 | **必须用 VARCHAR**（可含区间） |
| 6 | 功能边界 | 导入 + 管理台列表；报告统计衔接属后续 |
