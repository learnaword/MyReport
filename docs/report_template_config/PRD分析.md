# PRD 分析：报告模版配置

**需求来源：**  
1. 一段话 —「模版配置输出 JSON；含层级标题与指标…」  
2. 增量 —「可配置封面和底图」；入口「编辑样式」弹窗  
3. **再增量一段话** —「封面底图支持上传」  

**功能短名：** `report_template_config`  
**状态：** DRAFT — 仅需求  
**落地现状：** 标题/指标、编辑样式（填 URL/路径 + `saveStyle`）已实现；**本地文件上传尚未实现。**  
**关联现状：** 生成侧消费图片路径/URL（`strCoverImg` / `strBackCoverImg`）；Excel 导入已有 `MultipartFile` 模式可复用。

---

## 1. 问题陈述（上传增量）

「编辑样式」目前主要靠手填 URL/路径。运营侧图片多在本地，无现成可访问链接时无法落地封面/底图。

若不支持上传：外观配置门槛高，易退回外部传链或放弃配图。

---

## 2. 目标用户 / JTBD

**Primary：** 管理台配置人员（同既有模版管理）。  

**JTBD（上传）：**  
When 在「编辑样式」中配置封面或底图时，I want 直接上传本地 jpg/png，so I can 不依赖外链即可完成外观配置。

---

## 3. 成功标准（上传）

| 指标 | 目标 | 如何衡量 |
|------|------|----------|
| 封面可上传 | Must | 选文件上传成功后，详情 `coverImage` 有可回显引用 |
| 底图可上传 | Must | 同上，`backCoverImage` |
| 格式与大小受限 | Must | 非 jpg/png 或超 2MB 被拒绝并提示 |
| 上传后可清 | Must | 清除后库字段为空，界面无图 |

---

## 4. 方案概述（What）

在「编辑样式」弹窗中，封面、底图均支持：

1. **上传本地图片**（Must）→ 服务端落盘 → 回写可访问引用到模版字段  
2. **（可选保留）手填 URL/路径**（Should）→ 仍走现有 `saveStyle`  
3. **清除**（Must）

```text
编辑样式弹窗
├── 封面：【选择文件上传】 / 可选填 URL / 清空 / 预览（Could）
└── 底图：【选择文件上传】 / 可选填 URL / 清空 / 预览（Could）
```

库表仍只存**字符串引用**（路径或 URL），不存 BLOB。

## 6. 信息结构

```text
Template
├── coverImage        // 上传后的可访问路径或 URL
├── backCoverImage
└── nodes[]
```

落盘建议（产品约束，非实现定稿）：按模版区分目录，文件名防冲突；对外返回可被浏览器打开的路径（如 `/uploads/...` 或带文件服务前缀）。

---

## 7. 关键用户流程

1. 模版列表 → **编辑样式**  
2. 封面 / 底图区域点击 **上传**，选择本地 jpg/png（≤2MB）  
3. 上传成功 → 预览或显示路径 →（若需）点保存；或上传即落库  
4. 可清空；可改选手填 URL（若保留）  
5. `detail` 回显 `coverImage` / `backCoverImage`  

---

## 8. MVP 范围（上传增量）

**MVP：**

- 编辑样式弹窗：封面、底图均可上传 jpg/png，≤2MB  
- 校验失败有明确提示  
- 成功后字段可持久化并可再次打开回显  
- 可清除  

**可后置：**

- 缩略图精美预览、拖拽上传  
- 旧文件物理清理策略优化  
- OSS
