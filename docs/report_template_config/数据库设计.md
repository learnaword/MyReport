# 数据库设计：报告模版配置

**需求来源：** `docs/report_template_config/PRD分析.md`（含标题/指标及封面·底图增量）  
**功能短名：** `report_template_config`  
**目标库：** MySQL 8（与本仓库 `application.yml` / JPA `ddl-auto=update` 对齐）  
**设计原则：** 模版元数据（含外观）+ 邻接表存树；只存配置不存统计数据；导出 JSON 由应用层组装。

---

## 0. 已确认结论（来自 PRD）

| # | 议题 | 结论 | 库表影响 |
|---|------|------|----------|
| Q1–Q13 | 标题/指标树等 | 见历史；邻接表 + 软删 | 既有 `report_template` / `report_template_node` |
| Q18 | 外观入口 | 列表「编辑结构」后「编辑样式」弹窗配封面/底图 | **外观字段落在模版头表**，不进节点表 |
| Q17（默认） | 底图范围 | 仅报告级封底图，不含章节背景 | **不**在节点表加背景图列 |
| Q15（设计默认） | 资源形态 | 库内统一存**字符串引用**（URL 或日后上传后的路径均可） | `VARCHAR` 可空；不建上传文件表 |

---

## 1. 范围与实体

| 实体 | 表名 | 说明 |
|------|------|------|
| 报告模版 | `report_template` | 模版头：名称、说明、状态、**封面图、底图** |
| 模版节点 | `report_template_node` | 标题或指标；邻接表（不变） |

本期不建：独立「样式表」、图片资源表、章节背景字段、与 `createReport` 相关表。

**增量原则：** 封面/底图各至多一份，属模版级；可空表示未配置。

---

## 2. ER 关系

```text
report_template 1 ─── * report_template_node
report_template_node 1 ─── * report_template_node   （parent_id 自关联）
```

外观字段在 `report_template` 行内，无额外实体。

树约束：根 TITLE level=1；子 level=父+1；标题深度不限（TINYINT 实用上限约 127）；METRIC 为叶子。

---

## 3. 表结构

### 3.1 `report_template` — 报告模版（含外观增量）

| 列名 | 类型 | 空 | 默认 | 说明 |
|------|------|----|------|------|
| `id` | `BIGINT` | N | AI | 主键 |
| `name` | `VARCHAR(128)` | N | — | 模版名称 |
| `description` | `VARCHAR(512)` | Y | NULL | 模版说明 |
| `status` | `TINYINT` | N | `1` | `1`启用 / `0`停用 |
| `cover_image` | `VARCHAR(1024)` | Y | NULL | **封面图**引用（URL 或存储路径）；空=未配置 |
| `back_cover_image` | `VARCHAR(1024)` | Y | NULL | **底图**引用（报告级封底）；空=未配置 |
| `deleted` | `TINYINT` | N | `0` | 软删 |
| `delete_time` | `DATETIME` | Y | NULL | 软删时间 |
| `create_time` | `DATETIME` | N | — | |
| `update_time` | `DATETIME` | N | — | |

索引：`PRIMARY(id)`；`idx_status_update(status, update_time)`；`idx_deleted(deleted)`。

**说明：**

- 清除封面/底图 = 将对应列更新为 `NULL`（应用层勿长期存空串）  
- 复制模版时须**一并拷贝** `cover_image` / `back_cover_image`  
- `cover_text`（封面文字）见待确认 D6，**本期不加列**  

### 3.2 `report_template_node` — 不变

结构同前（`node_type` / `level` / 指标枚举 / 软删等）。**不**增加章节背景图列。

完整节点字段见历史版本要点：`template_id`、`parent_id`、`node_type`、`level`、`name`、`intro`、`sort_order`、`stat_field`、`value_format`、`display_type`、`chart_style`、软删与时间戳。

---

## 4. 枚举取值

节点侧枚举不变（`TITLE`/`METRIC`、`PERCENT`/`COUNT`、`TABLE`/`CHART`、`BAR`/`PIE`/`LINE`）。

外观侧无枚举：图片列为自由字符串。

---

## 5. 业务校验（应用层）

树规则不变。外观增量：

| 规则 | 说明 |
|------|------|
| 可选 | `cover_image` / `back_cover_image` 均可空 |
| 长度 | 单字段 ≤ 1024 |
| 无强制 URL 格式校验 | 允许相对路径；非法链接生成期再处理（本期不联调生成） |

---

## 6. 导出 JSON 逻辑形状（组装约定）

```json
{
  "templateId": 1,
  "name": "就业质量报告模版",
  "description": "",
  "status": 1,
  "coverImage": "https://example.com/cover.png",
  "backCoverImage": "https://example.com/back.png",
  "nodes": []
}
```

- 未配置时库内为 NULL；API 出参用 `null` 或 `""` 由 API 设计拍板  
- 与生成侧映射（非库表义务）：`coverImage` → `strCoverImg`，`backCoverImage` → `strBackCoverImg`（若 PRD Q20 改用旧名，仅影响 API 别名，**列名仍用上表 snake_case**）

## 7. DDL（参考）

### 7.1 全量（新库）

```sql
CREATE TABLE report_template (
  id                BIGINT        NOT NULL AUTO_INCREMENT COMMENT '主键',
  name              VARCHAR(128)  NOT NULL COMMENT '模版名称',
  description       VARCHAR(512)  DEFAULT NULL COMMENT '模版说明',
  status            TINYINT       NOT NULL DEFAULT 1 COMMENT '1启用 0停用',
  cover_image       VARCHAR(1024) DEFAULT NULL COMMENT '封面图URL或路径',
  back_cover_image  VARCHAR(1024) DEFAULT NULL COMMENT '底图URL或路径',
  deleted           TINYINT       NOT NULL DEFAULT 0 COMMENT '0未删 1已删',
  delete_time       DATETIME      DEFAULT NULL COMMENT '软删时间',
  create_time       DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  update_time       DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (id),
  KEY idx_status_update (status, update_time),
  KEY idx_deleted (deleted)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='报告模版';

CREATE TABLE report_template_node (
  id            BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键',
  template_id   BIGINT       NOT NULL COMMENT '模版ID',
  parent_id     BIGINT       DEFAULT NULL COMMENT '父节点ID，根为NULL',
  node_type     VARCHAR(16)  NOT NULL COMMENT 'TITLE / METRIC',
  level         TINYINT      NOT NULL COMMENT '树深度：根=1；子=父+1；标题深度不限',
  name          VARCHAR(256) NOT NULL COMMENT '标题名或指标名',
  intro         VARCHAR(2000) DEFAULT NULL COMMENT '标题介绍；指标为空',
  sort_order    INT          NOT NULL DEFAULT 0 COMMENT '同父排序',
  stat_field    VARCHAR(128) DEFAULT NULL COMMENT '数据集字段名（指标）',
  value_format  VARCHAR(16)  DEFAULT NULL COMMENT 'PERCENT / COUNT',
  display_type  VARCHAR(16)  DEFAULT NULL COMMENT 'TABLE / CHART',
  chart_style   VARCHAR(16)  DEFAULT NULL COMMENT 'BAR / PIE / LINE',
  deleted       TINYINT      NOT NULL DEFAULT 0 COMMENT '0未删 1已删',
  delete_time   DATETIME     DEFAULT NULL COMMENT '软删时间',
  create_time   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  update_time   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (id),
  KEY idx_template_id (template_id),
  KEY idx_parent_id (parent_id),
  KEY idx_template_parent_sort (template_id, parent_id, sort_order),
  KEY idx_node_deleted (deleted)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='报告模版节点（标题/指标邻接表）';
```

### 7.2 存量增量（已有 `report_template` 时）

```sql
ALTER TABLE report_template
  ADD COLUMN cover_image VARCHAR(1024) DEFAULT NULL COMMENT '封面图URL或路径' AFTER status,
  ADD COLUMN back_cover_image VARCHAR(1024) DEFAULT NULL COMMENT '底图URL或路径' AFTER cover_image;
```

> JPA `ddl-auto=update` 亦可自动加列；正式环境建议显式执行上文 ALTER。  
> 无数据库外键 CASCADE（D5）；软删与级联由应用层完成。

## 8. 与本仓库落地关系（设计备注）

| 层 | 落点（编码时） |
|----|----------------|
| Entity | `ReportTemplate` 增加 `coverImage` / `backCoverImage` |
| 复制 | `copyTemplate` 拷贝两列 |
| 详情 | `detail` 输出 `coverImage` / `backCoverImage` |
| 样式保存 | 新 API（如 `saveStyle`）只更新两列，不碰节点树 |
| 管理端 | 「编辑样式」弹窗 |

本文**不**写 API / 不编码。

## 9. 备选方案与取舍

| 方案 | 做法 | 结论 |
|------|------|------|
| A. 模版头两列（采用） | `cover_image` / `back_cover_image` | 一对一、简单，贴合「编辑样式」 |
| B. 独立 style 表 | `report_template_style` 1:1 | 过重，字段仅两个 |
| C. 塞进 JSON blob | `style_json` | 不如列直观；复制/回显更绕 |
