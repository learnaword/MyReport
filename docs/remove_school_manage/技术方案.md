# 技术方案：去掉学校管理（默认武汉大学）

**需求来源：** `docs/remove_school_manage/PRD分析.md`（一段话需求；文内 Q1–Q10 方括号结论**本期采用**）  
**功能短名：** `remove_school_manage`  
**状态：** DRAFT — 仅方案，不编码  
**目标：** 下线学校管理产品能力；就业导入、列表、报告创建/生成全链路固定归属「武汉大学」，用户无需选校。

---

## 0. 已确认结论（来自 PRD，影响方案）

| # | 议题 | 采用结论 | 方案影响 |
|---|------|----------|----------|
| Q1 | 旧学校导入入口 | **重定向到数据管理** `#data` | `GET /school/import` → `/admin/index.html#data` |
| Q2 | 报告列表学校列 | **固定展示「武汉大学」** | 可不调 `/school/list`；文案写死或回显默认校名 |
| Q3 | 默认校 ID | **按名称查找/自动创建，不强制固定 id** | `getOrCreateDefaultSchool()` |
| Q4 | Excel 学校列 | **忽略，强制归属武大** | 导入覆盖 `school_id`；不再按名称解析失败 |
| Q5 | 调用方仍传 schoolId | **忽略，强制默认校** | create/update/import 入参中的 schoolId 不参与决策 |
| Q6 | 报告唯一性 | **同模版仅一份未删报告** | 应用层按 `template_id` 判重（见 §5.3） |
| Q7 | `/school/*` API | **直接下线**（失败/不可用） | list/importExcel 返回失败；不保留只读兼容期 |
| Q8 | 历史非武大数据 | **MVP 列表只展示武大；清理可人工/后置** | 就业 list 默认按默认校过滤 |
| Q9 | 校名配置 | **写死「武汉大学」** | 常量，不引入环境变量 |
| Q10 | 文档 | **编码同次更新调用方文档** | `docs/admin_console.md` 等 |

## 1. 目标与非目标

### 1.1 目标

| # | 能力 |
|---|------|
| 1 | 管理台移除学校管理 panel；数据/报告去掉选校 UI |
| 2 | 内部统一「默认学校」解析：名称恒为「武汉大学」，不存在则自动插入一条 |
| 3 | 就业导入强制 `school_id = 默认校`；列表默认只查默认校 |
| 4 | 报告 create/update 不再依赖调用方 `schoolId`；生成聚合始终用默认校 |
| 5 | `/school/list`、`/school/importExcel` 等产品 API 下线；旧 import 入口重定向 `#data` |
| 6 | 同步更新对外 `docs/` 契约说明 |

### 1.2 非目标

- **不**物理删除 `school` 表、`School` 实体、`school_id` 列（保留内部归属键）
- **不**改就业宽表业务列、模版树、指标聚合算法、Spire 引擎
- **不**改 AI 简化报告主流程
- **不**做历史多校数据一键迁移工具（后置）
- **不**把默认校名做成可配置项（Q9）

## 2. 现状与切入点

| 能力 | 现状 | 本方案用法 |
|------|------|------------|
| 学校 CRUD/导入 | `SchoolController` + `SchoolService` + `SchoolExcelUtil` + admin `#school` | **产品下线**；保留 `SchoolRepository` 供 ensure 默认校 |
| 就业导入 | 可选 `schoolId`；Excel 解析学校名；无则**随机**选校 | 改为**始终**默认校；忽略入参与 Excel 学校列 |
| 就业列表 | 可选 `schoolId` 过滤 | 服务端默认强制按默认校过滤（忽略/覆盖查询参数） |
| 报告实例 | create 必填 `schoolId`；唯一 `school+template` | create 不收选校；写默认校；唯一改为**模版**维度 |
| 指标聚合 | `MetricAggregateService.aggregate(schoolId, …)` | 生成时传入默认校 id（或报告上已写死的默认校 id） |
| 管理台 | `loadSchoolOptions`、`mrSchool`、双下拉 | 删除学校 panel 与选校控件；报告列固定「武汉大学」 |

## 3. 总体架构

```text
                    ┌─────────────────────────────────┐
                    │ DefaultSchoolService（新建）      │
                    │  NAME = "武汉大学"                │
                    │  getOrCreate() → School.id       │
                    └───────────────┬─────────────────┘
                                    │
          ┌─────────────────────────┼─────────────────────────┐
          ▼                         ▼                         ▼
 GradEmploymentService      ManagedReportService      （可选）启动/首次调用
  import：强制 school_id     create/update：强制写入      预热一条 school
  list：按默认校过滤         generate：aggregate(默认校)
          │                         │
          ▼                         ▼
 grad_employment_record      managed_report.school_id
 （列结构不变）               （列保留，值恒为默认校）

 SchoolController
  GET  /school/import     → Redirect #data
  GET  /school/list       → code=-1「学校管理已下线」
  POST /school/importExcel→ 同上
  GET  /school/importProgress → 可保留或一并失败（无新任务即可）

 管理台 admin/index.html
  去掉 nav/panel-school；数据/报告去掉学校下拉；文案改流程
```

**原则：** 对外「无学校管理」；对内仍用 `school_id` 关联，单点 resolve，避免散落魔法数字。

## 4. 模块设计

### 4.1 DefaultSchoolService（新建）

**包：** `com.myreport.service`  
**常量：** `public static final String DEFAULT_SCHOOL_NAME = "武汉大学";`

**职责：**

| 方法 | 行为 |
|------|------|
| `getOrCreate()` | `findFirstByName(DEFAULT_SCHOOL_NAME)`；无则 `new School` 设 name 并 save；返回实体 |
| `requireId()` | 返回 `getOrCreate().getId()`（导入/建报告高频入口） |

**并发：** 两请求同时首次创建可能撞唯一名冲突（当前 `school.name` **无** DB 唯一约束）。MVP：catch 后再次 `findFirstByName`；若仍无则抛业务错。后置可对 `name` 加唯一索引（属数据库设计项）。

**不**提供 list/import 能力；原 `SchoolService` 的导入逻辑随产品下线可删或留死代码（推荐：Controller 下线后，`importFromExcel` 不再被调用，可删 Excel 导入路径以减负，见 §8）。

### 4.2 GradEmploymentService

**导入 `importFromExcel(..., Long schoolId, ...)`**

1. 开头：`Long defaultId = defaultSchoolService.requireId();`  
2. **忽略**参数 `schoolId`（Q5）  
3. 解析行后：对每行 **无条件** `setSchoolId(defaultId)`（Q4）  
4. 删除/旁路：`resolveSchoolId` 按名称匹配、`pickRandomSchoolId` 随机选校  
5. Excel 仍可含「学校名称/学校ID」列：读入后被覆盖，不因此失败  

**唯一键不变：** `(school_id, student_no, graduation_year)` — 全部落到同一默认校后，同学号同届覆盖更新，符合单校预期。

**列表 `list(page, size, schoolId)`**

- 改为始终 `findBySchoolId(defaultSchoolService.requireId(), pageable)`  
- 查询参数 `schoolId` **忽略**（与 Q5/Q8 一致）  
- Controller 可保留参数以免前端旧脚本炸，但文档标明已忽略  

**`validateDefaultSchoolId`：** 可改为空操作或删除调用（不再校验调用方传入的学校）。

### 4.3 ManagedReportService

**创建 `create(name, schoolId, templateId)`**

- 签名可保留 `schoolId` 以减 Controller 改动面，但**忽略**入参  
- `Long sid = defaultSchoolService.requireId();` 写入  
- 唯一校验改为：`existsByTemplateIdAndDeleted(templateId, NOT_DELETED)`（需 Repository 新方法）  
- 冲突文案：「该模版已存在报告」  

**更新 `update(...)`**

- 忽略入参 `schoolId`；若需纠正历史，可将 `r.setSchoolId(defaultId)` 在每次 update 时写回默认校（推荐：update/generate 前校正，避免旧报告仍指向他校）  
- 模版变更时按「新 template_id」做唯一校验（`idNot`）  
- `GENERATING` 时仍拒绝改 `template_id`（沿用现规则；学校不可改）  

**生成 `generate`**

- `requireSchool` 改为确保默认校存在，或直接用 `r.getSchoolId()`（若已校正为默认校）  
- `metricAggregateService.aggregate(defaultId 或 r.getSchoolId(), …)`  
- `CreateReportVO.lSchoolId` 继续传默认校 id（引擎侧若未强依赖，保持兼容）  

**列表 `toListItem`**

- `schoolName` 固定返回 `"武汉大学"`（或 `defaultSchoolService` 的常量），避免依赖他校行上的错误名称（Q2）  
- 仍可返回 `schoolId`（默认校真实 id）供调试；管理台可不展示 id  

### 4.4 ManagedReportRepository

新增：

```text
boolean existsByTemplateIdAndDeleted(Long templateId, Integer deleted);
boolean existsByTemplateIdAndDeletedAndIdNot(Long templateId, Integer deleted, Long id);
```

原 `existsBySchoolIdAndTemplateIdAndDeleted*` 可弃用。

**库唯一索引：** 现有设计含生成列 `uk_active_school_template`（`school_id:template_id`）。在「学校恒为武大」后，与「模版唯一」在**新数据**上近似等价；但若库中仍有「他校 + 同模版」未删行，应用层模版唯一会拒绝新建，而 DB 生成列仍允许「武大 + 同模版」——以**应用层为准**即可。生成列改为 `active_template` 属可选加固，放到「做数据库设计」时再定；本期可不改表。

### 4.5 SchoolController（产品下线）

| 方法 | 路径 | 行为 |
|------|------|------|
| GET | `/school/import` | `RedirectView("/admin/index.html#data")` |
| GET | `/school/list` | `{ code: -1, message: "学校管理已下线" }` |
| POST | `/school/importExcel` | 同上 |
| GET | `/school/importProgress` | 同上，或「任务不存在」；无新写入即可 |

静态页 `src/main/resources/static/school/import.html`：改为跳转 `#data` 或删除并由网关/资源 404；推荐页内 `location` 到 `/admin/index.html#data`。

`SchoolService.importFromExcel` / `SchoolExcelUtil`：无引用后可删（减维护）；若希望 diff 更小，可暂留死代码，Task 中标「可删」。

### 4.6 GradEmploymentController / ManagedReportController

| 接口 | 变更 |
|------|------|
| `GET /grad-employment/list?schoolId=` | 参数保留但**忽略**；服务端按默认校过滤 |
| `POST /grad-employment/importExcel` | `schoolId` 保留但**忽略** |
| `POST /managed-report/create` | `schoolId` **不再必填**；有则忽略 |
| `POST /managed-report/update` | `schoolId` 可选；有则忽略 |
| 列表响应 | `schoolName` 恒为「武汉大学」 |

校验：create 仅校验 `name`、`templateId`；缺 `schoolId` 不再报错。

### 4.7 管理台 `admin/index.html`

| 项 | 改动 |
|----|------|
| 侧栏 | 删除「学校管理」按钮 |
| `#panel-school` | 删除整段 section（或 hidden 且无入口，推荐删除） |
| 首页引导 | 「学校→数据→…」改为「数据→模版→报告」；文案去掉「管理学校」 |
| 数据管理 | 删除 `#dataSchoolFilter`、`#dataSchoolImport`；导入不再附带 schoolId；列表列「学校」可改为固定文案或删除列 |
| 报告管理 | 删除 `#mrSchool` 与表头「学校」列，或学校列写死「武汉大学」；create/update body **不传** schoolId |
| JS | 删除 `loadSchools` / `loadSchoolOptions` / `schoolLabel` 依赖链；hash 白名单去掉 `school`；`#school` 访问可落到 `#data` 或 intro |
| 初始化 | 去掉启动时为填下拉而调 `/school/list` |

## 5. 关键数据流

### 5.1 就业导入

```text
POST /grad-employment/importExcel (file[, schoolId 忽略])
  → DefaultSchoolService.requireId()
  → 解析 Excel 行
  → 每行 school_id := 默认校 id（覆盖 Excel 学校列）
  → upsert by (school_id, student_no, graduation_year)
```

### 5.2 报告创建与生成

```text
POST /managed-report/create { name, templateId }
  → requireId() → school_id 落库
  → 校验 template 唯一（未删）
  → save

POST /managed-report/generate { id }
  → 校正 school_id 为默认校（可选但推荐）
  → aggregate(默认校 id, metrics)
  → SpireReportUtil.createReport(..., lSchoolId=默认校 id)
```

## 6. 历史数据策略（MVP）

| 数据 | 策略 |
|------|------|
| `school` 多行 | 保留；确保存在「武汉大学」一行即可 |
| `grad_employment_record` 非武大 `school_id` | **列表不可见**（只查默认校）；不自动 UPDATE 迁移 |
| `managed_report` 非武大 | 列表仍可见（报告管理未按校过滤）；`schoolName` 展示仍为「武汉大学」；**generate/update 时写回默认校 id**，避免聚合仍扫旧校空数据 |

若运营需要把旧就业行并入武大：后置手工 SQL / 小工具（非本期）。

## 7. 风险与缓解

| 风险 | 缓解 |
|------|------|
| 首次并发创建「武汉大学」重复行 | getOrCreate 冲突后重查；后置 name 唯一索引 |
| 旧报告 school_id 指向他校 → 生成空表 | generate/update 校正为默认校 |
| 外部仍调 `/school/list` | 明确 -1 + 文档；管理台不再依赖 |
| DB `uk_active_school_template` 与「模版唯一」不完全一致 | 应用层模版唯一；库索引变更后置 |
| 就业唯一键在「迁到武大」前同学号分属两校 | MVP 不迁移则无冲突；若后置迁移需先处理冲突学号 |

## 8. 实现顺序建议（供拆 Task，非本项产出）

1. `DefaultSchoolService` + 就业强制默认校 + list 过滤  
2. `ManagedReport` 无选校创建/更新/生成校正 + Repository 模版唯一  
3. `SchoolController` 下线与重定向  
4. 管理台 UI/文案  
5. `docs/` + `AGENTS.md`  
6. （可选）删除 `SchoolExcelUtil` / 无用导入代码  

验证：`make build`；手动：空库启动 → 直接导就业 → 建模版 → 建报告（无选校）→ 生成有数据的 Word；访问 `#school` / `/school/list` 应符合下线行为。
