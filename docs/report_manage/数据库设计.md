# 数据库设计：报告管理

**需求来源：** `docs/report_manage/PRD分析.md`（含文内已标注的 Q1–Q15 结论）  
**功能短名：** `report_manage`  
**目标库：** MySQL 8（与本仓库 `application.yml` / JPA `ddl-auto=update` 对齐）  
**设计原则：** 只持久化**报告实例**与生成结果引用；模版树与指标元数据只读既有表；统计结果**不落库**（生成时现算）；无库表外键 CASCADE。

---

## 0. 已确认结论（来自 PRD，影响库表）

| # | 议题 | 结论 | 库表影响 |
|---|------|------|----------|
| Q1 | 是否持久化报告实例 | **是** | 新建报告实例表 |
| Q2 | 同校 + 同模版多份 | **不允许** | 未删记录上 `(school_id, template_id)` 唯一 |
| Q3 | 改学校/模版 | **允许；改后不清空旧文件** | 可更新外键列；`file_path` 保留至下次成功覆盖或删除 |
| Q4 | 删报告时删 docx | **删文件** | 库软删 + 应用层删磁盘；库内清/保留路径见 §5 |
| Q5 | 生成锁 / reportId | **沿用现有 reportId** | 本表主键 `id` **即** `reportId`（Redis / `createReport`） |
| Q6 | 届次过滤 | **不过滤** | 报告表**不**加 `graduation_year` |
| Q7–Q13 | 聚合口径 / JSON 形态 | 应用层约定 | **不**建指标结果表 |
| Q14 | 下载形态 | **落盘路径写 Redis/库** | 表内存 `file_path` |
| Q15 | 模版停用后生成 | **不允许** | 生成时读 `report_template.status`（不冗余拷贝到本表） |

其余（PERCENT 格式、跳过失败节点等）属计算逻辑，无独立表。

## 1. 范围与实体

| 实体 | 表名 | 说明 |
|------|------|------|
| 报告实例 | `managed_report` | 管理台创建的报告：名称、学校、模版、生成状态、文件路径 |
| 学校（既有） | `school` | 报告必选；应用层校验存在 |
| 报告模版（既有） | `report_template` | 报告必选；生成时只读树与外观 |
| 模版节点（既有） | `report_template_node` | 指标元数据；不改表 |
| 就业去向（既有） | `grad_employment_record` | 指标聚合数据源；不改表 |

**本期新建：** 仅 `managed_report`。

**本期不建：**

| 项 | 原因 |
|----|------|
| 指标结果表 / 聚合缓存表 | 生成时现算；MVP 不预聚合 |
| `reportJsonArr` 快照表 | 可重算；避免大 JSON 落库（调试若需要可后置） |
| 生成历史版本表 | 未要求；同实例覆盖路径即可 |
| 报告–模版节点关联表 | 整模版绑定，非点选节点 |

## 2. ER 关系

```text
school            1 ─── * managed_report
report_template   1 ─── * managed_report

report_template   1 ─── * report_template_node   （既有，生成只读）
school            1 ─── * grad_employment_record （既有，聚合只读）
```

逻辑关系（**无物理外键**）：

- `managed_report.school_id` → `school.id`
- `managed_report.template_id` → `report_template.id`
- `managed_report.id` ≡ 生成侧 / Redis 的 `reportId`

## 3. 表结构

### 3.1 `managed_report` — 报告实例

| 列名 | 类型 | 空 | 默认 | 说明 |
|------|------|----|------|------|
| `id` | `BIGINT` | N | AI | 主键；**同时作为** `createReport` / Redis 的 `reportId` |
| `name` | `VARCHAR(128)` | N | — | 报告名称 |
| `school_id` | `BIGINT` | N | — | 学校 ID |
| `template_id` | `BIGINT` | N | — | 模版 ID |
| `generate_status` | `TINYINT` | N | `0` | 生成状态，见 §4 |
| `file_path` | `VARCHAR(1024)` | Y | NULL | 最近一次**成功**生成的 `.docx` 落盘路径；未成功为空 |
| `fail_message` | `VARCHAR(512)` | Y | NULL | 最近一次失败原因（对外可读摘要）；成功时清空 |
| `last_generate_time` | `DATETIME` | Y | NULL | 最近一次进入「生成中」或结束的时间（应用写入） |
| `deleted` | `TINYINT` | N | `0` | `0` 未删 / `1` 已删 |
| `delete_time` | `DATETIME` | Y | NULL | 软删时间 |
| `create_time` | `DATETIME` | N | — | |
| `update_time` | `DATETIME` | N | — | |

**生成列（唯一约束用）：**

| 列名 | 类型 | 说明 |
|------|------|------|
| `active_school_template` | `VARCHAR(64)` | `STORED GENERATED`：未删时为 `CONCAT(school_id, ':', template_id)`，已删为 `NULL` |

索引：

| 名 | 列 | 说明 |
|----|-----|------|
| `PRIMARY` | `id` | |
| `uk_active_school_template` | `active_school_template` | **唯一**；`NULL` 不冲突 → 软删后可再建同校同模版 |
| `idx_school_id` | `school_id` | 按校列表 |
| `idx_template_id` | `template_id` | 按模版反查 |
| `idx_status_update` | `generate_status, update_time` | 列表筛选 |
| `idx_deleted` | `deleted` | 默认过滤未删 |

**说明：**

- 不设 DB 外键；创建/更新时应用校验：学校存在；模版存在且 `deleted=0`；创建时模版宜 `status=1`（生成时再强制 Q15）。
- `file_path`：成功生成后写入/覆盖；按 Q3 **改学校/模版不清空**本列（可能短暂指向旧文件，下次成功再覆盖；删除实例时应用删盘并建议置 `NULL`）。
- `fail_message`：勿写入堆栈或本机绝对机密路径细节；内部日志另记。
- 与 `CreateReportVO.reportId`（当前 `Integer`）：实务上自增 `id` 远小于 `Integer.MAX_VALUE`；应用传入时用 `id` 作为 `reportId`。若未来突破 int，需同步放宽 VO / Redis 格式（非本期表结构问题）。

## 4. 枚举取值

### 4.1 `generate_status`

| 值 | 含义 | 说明 |
|----|------|------|
| `0` | `DRAFT` | 已创建，尚未成功生成（含从未点生成） |
| `1` | `GENERATING` | 生成中（已占 `REPORT_LOCK`） |
| `2` | `SUCCESS` | 最近一次生成成功，`file_path` 应非空 |
| `3` | `FAILED` | 最近一次生成失败，`fail_message` 宜非空 |

状态迁移（应用层，非 DB 约束）：

```text
DRAFT ──生成──► GENERATING ──成功──► SUCCESS
                      │
                      └──失败──► FAILED
SUCCESS / FAILED / DRAFT ──再次生成──► GENERATING（若未持锁）
```

软删后查询默认不可见；`generate_status` 不必改为终态。

### 4.2 不入库的计算约定（对照，免建表）

| 项 | 约定 |
|----|------|
| 数据源 | `grad_employment_record`，按 `school_id`，**不按届次过滤** |
| 分组 | `statField` 对应列的取值 |
| COUNT | 各组行数 |
| PERCENT | 组计数 / 该校该字段非空总人数；展示 2 位小数 + `%` |
| 失败指标 | 跳过节点继续 |

## 5. 业务校验（应用层）

| 规则 | 说明 |
|------|------|
| 创建必填 | `name`、`school_id`、`template_id` |
| 名称长度 | ≤ 128 |
| 唯一 | 未删记录下同一 `school_id` + `template_id` 仅一条（依赖 `uk_active_school_template`） |
| 更新 | 允许改 `name` / `school_id` / `template_id`；若新组合与另一未删行冲突则失败 |
| 生成中改配 | **建议** `GENERATING` 时拒绝改学校/模版（避免边生成边换数）；PRD 允许改配指非生成中场景。若产品坚持生成中也可改，须文档说明竞态，**默认拒绝** |
| 生成前 | 模版未删且 `status=1`；学校存在 |
| 删除 | `deleted=1` + `delete_time`；尽力删除 `file_path` 对应文件后将 `file_path` 置 `NULL` |
| 进度 | Redis `report:list:{id}` / `report:lock:{id}`；库内 `generate_status` 与终态对齐，避免仅 Redis 可查 |

## 7. DDL（参考）

### 7.1 全量（新表）

```sql
CREATE TABLE managed_report (
  id                      BIGINT         NOT NULL AUTO_INCREMENT COMMENT '主键，兼作生成侧 reportId',
  name                    VARCHAR(128)   NOT NULL COMMENT '报告名称',
  school_id               BIGINT         NOT NULL COMMENT '学校ID',
  template_id             BIGINT         NOT NULL COMMENT '模版ID',
  generate_status         TINYINT        NOT NULL DEFAULT 0 COMMENT '0草稿 1生成中 2成功 3失败',
  file_path               VARCHAR(1024)  DEFAULT NULL COMMENT '成功生成的docx路径',
  fail_message            VARCHAR(512)   DEFAULT NULL COMMENT '最近失败原因摘要',
  last_generate_time      DATETIME       DEFAULT NULL COMMENT '最近生成相关时间',
  deleted                 TINYINT        NOT NULL DEFAULT 0 COMMENT '0未删 1已删',
  delete_time             DATETIME       DEFAULT NULL COMMENT '软删时间',
  create_time             DATETIME       NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  update_time             DATETIME       NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  active_school_template  VARCHAR(64)
    GENERATED ALWAYS AS (
      IF(deleted = 0, CONCAT(school_id, ':', template_id), NULL)
    ) STORED COMMENT '未删时校+模版唯一键；已删为NULL',
  PRIMARY KEY (id),
  UNIQUE KEY uk_active_school_template (active_school_template),
  KEY idx_school_id (school_id),
  KEY idx_template_id (template_id),
  KEY idx_status_update (generate_status, update_time),
  KEY idx_deleted (deleted)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='报告实例（管理台报告管理）';
```

### 7.2 JPA 注意

- `ddl-auto=update` **不一定**可靠创建 `GENERATED ... STORED` 列与唯一约束；正式环境建议执行上文 DDL。
- 若暂不使用生成列：可改为应用层唯一校验 + 普通索引 `idx_school_template_deleted (school_id, template_id, deleted)`，并在文档中标明竞态窗口；**推荐仍用生成列唯一约束**。

## 10. 风险

| 风险 | 可能性 | 影响 | 缓解 |
|------|--------|------|------|
| JPA 未建出生成列 | 中 | 唯一约束缺失，出现重复校+模版 | 初始化 SQL / 显式 DDL |
| `id` 与历史外部传入的 `reportId` 撞号 | 低 | Redis 锁/进度串扰 | 管理台报告自增段独立；外部仍自带 id 时分区或统一入口 |
| 改配后 `file_path` 与当前校/模版不一致 | 中 | 下载到旧稿 | 列表展示「文件对应上次成功」；或 UI 提示已改配未重生成 |
| 无 FK 导致孤儿 school/template | 低 | 展示名缺失 | 列表 join 时降级显示 ID；删除学校前业务约束后置 |
