# 数据库设计：AI 简化报告

**需求来源：** `docs/ai_simple_report/PRD分析.md`、`docs/ai_simple_report/技术方案.md`  
**功能短名：** `ai_simple_report`  
**目标库：** MySQL 8（与本仓库 `application.yml` / JPA `ddl-auto=update` 对齐）  
**设计原则：** 配置与运行实例**独立成表**，不共用 `managed_report`；拉数结果与 `reportJsonArr` **不落库**（现算）；交付以路径字段记录；无物理外键 CASCADE；`engine_report_id` 与 managed 报告 id 段隔离。

---

## 0. 已确认结论（影响库表）

| 来源 | 议题 | 结论 | 库表影响 |
|------|------|------|----------|
| Q10 | 实体归属 | **独立**简化报告 | 新建三表，不改 `managed_report` |
| Q1 | 配置入口 | 独立管理台 | 配置表可独立 CRUD |
| Q8 | 交付 | MVP **仅目录** | `delivery_dir` + Run 上 `delivery_path`；不做邮件表 |
| Q6 | 确认 | 人工确认后出 Word | Run 状态含 `PENDING_CONFIRM` |
| Q9 | 定时 | Quartz | 配置表预留 `schedule_enabled` / `cron_expr` |
| Q2 | 鉴权 | MVP 无 | 块表**不**存 token 列 |
| Q4 | 样式 | TABLE/BAR/PIE/LINE | `render_style` 枚举字符串 |
| 技术方案 §6.1 | Redis reportId | `BASE + run.id` | Run 存 `engine_report_id` |
| 技术方案 §4.1 | 配置快照 | 防确认后改配漂移 | Run 存 `config_snapshot_json` |

## 1. 范围与实体

| 实体 | 表名 | 说明 |
|------|------|------|
| 简化报告配置 | `simple_report` | 名称、交付目录、启用、定时、软删 |
| 数据区块 | `simple_report_block` | 一报告多接口；有序；HTTP + 渲染样式 |
| 运行实例 | `simple_report_run` | 一次 plan→confirm→生成；MD、快照、状态、路径 |

**本期新建：** 上述三表。

**本期不建：**

| 项 | 原因 |
|----|------|
| 接口拉数结果表 / 图表缓存 | 生成时现拉现算 |
| `reportJsonArr` 快照表 | 可按快照重跑；避免大 JSON 双份（调试后置） |
| 邮件投递队列表 | MVP 不做邮件 |
| 与 `managed_report` 关联表 | 产品隔离 |
| Quartz 官方表（若用 `@Scheduled`） | 可选；若上完整 Quartz JDBC JobStore 再用官方 DDL，**非业务表** |

**不修改：** `managed_report`、`report_template*`、`school`、`grad_employment_record`。

## 2. ER 关系

```text
simple_report  1 ─── * simple_report_block
simple_report  1 ─── * simple_report_run
```

逻辑关系（**无物理外键**）：

- `simple_report_block.report_id` → `simple_report.id`
- `simple_report_run.report_id` → `simple_report.id`
- `simple_report_run.engine_report_id` → 生成侧 / Redis 的 `reportId`（**不是**本表主键；见 §4.3）

删除配置：软删 `simple_report`；块可级联软删或硬删未运行关联块（应用层）；历史 Run **保留**（审计）。

## 3. 表结构

### 3.1 `simple_report` — 简化报告配置

| 列名 | 类型 | 空 | 默认 | 说明 |
|------|------|----|------|------|
| `id` | `BIGINT` | N | AI | 主键 |
| `name` | `VARCHAR(128)` | N | — | 报告名称 |
| `status` | `TINYINT` | N | `1` | `1` 启用 / `0` 停用；停用后不可 plan/定时 |
| `delivery_dir` | `VARCHAR(512)` | N | — | 成功后复制目录（相对 delivery-root 或经校验的路径）；禁止 `..` |
| `schedule_enabled` | `TINYINT` | N | `0` | `0` 关 / `1` 开（Quartz） |
| `cron_expr` | `VARCHAR(64)` | Y | NULL | 启用定时时必填（应用校验）；如 `0 0 9 * * ?` |
| `deleted` | `TINYINT` | N | `0` | `0` 未删 / `1` 已删 |
| `delete_time` | `DATETIME` | Y | NULL | 软删时间 |
| `create_time` | `DATETIME` | N | — | |
| `update_time` | `DATETIME` | N | — | |

**生成列（名称唯一，软删可复用）：**

| 列名 | 类型 | 说明 |
|------|------|------|
| `active_name` | `VARCHAR(128)` | `STORED GENERATED`：未删时为 `name`，已删为 `NULL` |

索引：

| 名 | 列 | 说明 |
|----|-----|------|
| `PRIMARY` | `id` | |
| `uk_active_name` | `active_name` | 未删名称唯一 |
| `idx_status_deleted` | `status, deleted` | 列表 |
| `idx_schedule` | `schedule_enabled, deleted` | 定时扫描 |
| `idx_update_time` | `update_time` | |

**说明：**

- 名称唯一仅约束未删行；软删后允许同名再建。
- `delivery_dir` 存**配置值**（相对路径片段或业务约定路径）；是否落在白名单根由应用校验，库不存绝对机密盘符亦可。
- MVP 不加 `creator`。

### 3.2 `simple_report_block` — 数据区块（多接口）

| 列名 | 类型 | 空 | 默认 | 说明 |
|------|------|----|------|------|
| `id` | `BIGINT` | N | AI | 主键 |
| `report_id` | `BIGINT` | N | — | 所属 `simple_report.id` |
| `sort_order` | `INT` | N | `0` | 文档区块顺序，从小到大 |
| `title` | `VARCHAR(128)` | N | — | 区块标题（写入 Word 指标标题） |
| `http_method` | `VARCHAR(8)` | N | `GET` | `GET` / `POST` |
| `url` | `VARCHAR(1024)` | N | — | 完整 URL（http/https）；应用做 SSRF 白名单 |
| `query_json` | `TEXT` | Y | NULL | 查询参数 JSON 对象，可空 |
| `body_json` | `TEXT` | Y | NULL | POST body JSON，可空；GET 应为空 |
| `render_style` | `VARCHAR(16)` | N | — | `TABLE` / `BAR` / `PIE` / `LINE` |
| `prompt_hint` | `VARCHAR(512)` | Y | NULL | 给计划 MD「查询目的」的提示 |
| `create_time` | `DATETIME` | N | — | |
| `update_time` | `DATETIME` | N | — | |

索引：

| 名 | 列 | 说明 |
|----|-----|------|
| `PRIMARY` | `id` | |
| `idx_report_sort` | `report_id, sort_order` | 按报告加载有序块 |
| `idx_report_id` | `report_id` | |

**说明：**

- **不设** `deleted`：改配以「整表替换 blocks」或按 id 增删为主；历史以 Run 快照为准。
- **不存**鉴权头/Token（Q2）。
- 同一 `report_id` 下 `sort_order` 建议唯一（应用保证；可选后续加唯一索引）。
- `url` 长度 1024；超长查询串应放 `query_json`。

### 3.3 `simple_report_run` — 运行实例

| 列名 | 类型 | 空 | 默认 | 说明 |
|------|------|----|------|------|
| `id` | `BIGINT` | N | AI | 主键（运行号） |
| `report_id` | `BIGINT` | N | — | 发起时的配置 id |
| `report_name` | `VARCHAR(128)` | N | — | 冗余当时名称，列表展示用 |
| `user_prompt` | `VARCHAR(2000)` | Y | NULL | 用户诉求原文；定时可为固定文案 |
| `plan_md` | `MEDIUMTEXT` | Y | NULL | 给人审的计划 Markdown |
| `config_snapshot_json` | `MEDIUMTEXT` | N | — | 创建 plan 时序列化的配置快照（含 blocks）；**执行只认此快照** |
| `run_status` | `TINYINT` | N | — | 见 §4.1 |
| `engine_report_id` | `INT` | Y | NULL | 传入 Spire/Redis 的 `reportId`；confirm 时分配 |
| `warn_count` | `INT` | N | `0` | 区块拉取失败但占位继续的次数 |
| `file_path` | `VARCHAR(1024)` | Y | NULL | Spire 成功落盘路径（引擎侧） |
| `delivery_path` | `VARCHAR(1024)` | Y | NULL | 复制到 `delivery_dir` 后的完整路径 |
| `fail_message` | `VARCHAR(512)` | Y | NULL | 失败摘要；成功宜清空 |
| `trigger_type` | `TINYINT` | N | `0` | `0` 人工 / `1` 定时 |
| `confirmed_time` | `DATETIME` | Y | NULL | 确认执行时间 |
| `finished_time` | `DATETIME` | Y | NULL | 进入 SUCCESS/FAILED 时间 |
| `create_time` | `DATETIME` | N | — | plan 创建时间 |
| `update_time` | `DATETIME` | N | — | |

索引：

| 名 | 列 | 说明 |
|----|-----|------|
| `PRIMARY` | `id` | |
| `uk_engine_report_id` | `engine_report_id` | 非空唯一，避免两 Run 共用同一 Redis id |
| `idx_report_status` | `report_id, run_status` | 按配置查运行 |
| `idx_status_create` | `run_status, create_time` | 待确认列表 |
| `idx_create_time` | `create_time` | |

**`config_snapshot_json` 建议形态（逻辑，非另表）：**

```json
{
  "reportId": 1,
  "name": "周报简化",
  "deliveryDir": "simple/weekly",
  "blocks": [
    {
      "title": "就业率",
      "httpMethod": "GET",
      "url": "https://example.com/api/x",
      "queryJson": null,
      "bodyJson": null,
      "renderStyle": "BAR",
      "promptHint": "按学院"
    }
  ]
}
```

**`engine_report_id` 分配：**

```text
BASE = 1000000000
engine_report_id = BASE + run.id     // 须 ≤ Integer.MAX_VALUE (2147483647)
→ run.id 理论上限约 1_147_483_647；实务远小于此
```

确认执行前写入；未确认前为 `NULL`（不占 Redis）。

## 4. 枚举取值

### 4.1 `simple_report_run.run_status`

| 值 | 常量名 | 说明 |
|----|--------|------|
| `1` | `PENDING_CONFIRM` | 计划已生成，待人工确认 |
| `2` | `GENERATING` | 已确认，拉数/Spire 进行中 |
| `3` | `SUCCESS` | 成功；`file_path` / `delivery_path` 宜非空 |
| `4` | `FAILED` | 失败；`fail_message` 宜非空 |
| `5` | `CANCELLED` | 可选：确认前作废 |

技术方案中的 `PLANNING`：**不落库**（同步生成 MD 后直接 `PENDING_CONFIRM`）。

状态迁移（应用层）：

```text
（POST plan）──► PENDING_CONFIRM ──confirm──► GENERATING ──► SUCCESS
                      │                          │
                      └── cancel ──► CANCELLED     └──► FAILED

仅 PENDING_CONFIRM 可 confirm；GENERATING 不可重复 confirm。
```

定时策略 A（技术方案默认）：Job 只创建 `PENDING_CONFIRM`，**不**自动进入 `GENERATING`。

### 4.2 `simple_report.status`

| 值 | 含义 |
|----|------|
| `0` | 停用 |
| `1` | 启用 |

### 4.3 `render_style`

| 值 | 生成侧映射 |
|----|------------|
| `TABLE` | `displayType=TABLE` |
| `BAR` | `CHART` + `BAR` |
| `PIE` | `CHART` + `PIE` |
| `LINE` | `CHART` + `LINE` |

### 4.4 `http_method`

| 值 |
|----|
| `GET` |
| `POST` |

### 4.5 `trigger_type`

| 值 | 含义 |
|----|------|
| `0` | `MANUAL` |
| `1` | `SCHEDULE` |

## 5. 业务校验（应用层）

| 规则 | 说明 |
|------|------|
| 创建报告 | `name`、`delivery_dir` 必填；名称未删唯一 |
| 区块 | 至少 1 个才能 plan；`url`、`title`、`render_style` 必填 |
| URL | 仅 http/https；host ∈ 白名单；禁回环/元数据地址 |
| `delivery_dir` | 无 `../`；落在配置的 delivery-root 下 |
| 定时 | `schedule_enabled=1` 时 `cron_expr` 非空且可解析 |
| plan | 报告 `status=1` 且未删；写入快照 + MD → `PENDING_CONFIRM` |
| confirm | 仅 `PENDING_CONFIRM`；分配 `engine_report_id`；置 `GENERATING` |
| 成功 | 写 `file_path`、`delivery_path`、`finished_time`；`warn_count` 可 >0 |
| 全块失败 | `FAILED`（与技术方案 Q7 默认一致） |
| 删配置 | 软删；不停历史 Run；块可物理删除或随配置一并逻辑忽略 |
| `fail_message` | 不写堆栈/密钥 |

## 7. DDL（参考）

### 7.1 `simple_report`

```sql
CREATE TABLE simple_report (
  id                 BIGINT        NOT NULL AUTO_INCREMENT COMMENT '主键',
  name               VARCHAR(128)  NOT NULL COMMENT '报告名称',
  status             TINYINT       NOT NULL DEFAULT 1 COMMENT '1启用 0停用',
  delivery_dir       VARCHAR(512)  NOT NULL COMMENT '交付目录（相对白名单根）',
  schedule_enabled   TINYINT       NOT NULL DEFAULT 0 COMMENT '0关 1开',
  cron_expr          VARCHAR(64)   DEFAULT NULL COMMENT 'Quartz cron',
  deleted            TINYINT       NOT NULL DEFAULT 0 COMMENT '0未删 1已删',
  delete_time        DATETIME      DEFAULT NULL COMMENT '软删时间',
  create_time        DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  update_time        DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  active_name        VARCHAR(128)
    GENERATED ALWAYS AS (IF(deleted = 0, name, NULL)) STORED
    COMMENT '未删时名称唯一；已删为NULL',
  PRIMARY KEY (id),
  UNIQUE KEY uk_sr_active_name (active_name),
  KEY idx_sr_status_deleted (status, deleted),
  KEY idx_sr_schedule (schedule_enabled, deleted),
  KEY idx_sr_update_time (update_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='AI简化报告配置';
```

### 7.2 `simple_report_block`

```sql
CREATE TABLE simple_report_block (

```

### 7.3 `simple_report_run`

```sql
CREATE TABLE simple_report_run (
  id                     BIGINT         NOT NULL AUTO_INCREMENT COMMENT '主键',
  report_id              BIGINT         NOT NULL COMMENT 'simple_report.id',
  report_name            VARCHAR(128)   NOT NULL COMMENT '当时报告名冗余',
  user_prompt            VARCHAR(2000)  DEFAULT NULL COMMENT '用户诉求',
  plan_md                MEDIUMTEXT     DEFAULT NULL COMMENT '计划Markdown',
  config_snapshot_json   MEDIUMTEXT     NOT NULL COMMENT '执行用配置快照JSON',
  run_status             TINYINT        NOT NULL COMMENT '1待确认 2生成中 3成功 4失败 5取消',
  engine_report_id       INT            DEFAULT NULL COMMENT 'Spire/Redis reportId = BASE+id',
  warn_count             INT            NOT NULL DEFAULT 0 COMMENT '区块失败占位次数',
  file_path              VARCHAR(1024)  DEFAULT NULL COMMENT '引擎docx路径',
  delivery_path          VARCHAR(1024)  DEFAULT NULL COMMENT '交付目录中的路径',
  fail_message           VARCHAR(512)   DEFAULT NULL COMMENT '失败摘要',
  trigger_type           TINYINT        NOT NULL DEFAULT 0 COMMENT '0人工 1定时',
  confirmed_time         DATETIME       DEFAULT NULL COMMENT '确认时间',
  finished_time          DATETIME       DEFAULT NULL COMMENT '结束时间',
  create_time            DATETIME       NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  update_time            DATETIME       NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (id),
  UNIQUE KEY uk_srr_engine_report_id (engine_report_id),
  KEY idx_srr_report_status (report_id, run_status),
  KEY idx_srr_status_create (run_status, create_time),
  KEY idx_srr_create_time (create_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='AI简化报告运行实例';
```

### 7.4 JPA 注意

- `ddl-auto=update` **不一定**可靠创建 `GENERATED ... STORED` 与 `MEDIUMTEXT` 细节；正式环境建议执行上文 DDL。
- `engine_report_id` 用 `INT` 对齐 `CreateReportVO.reportId`（Integer）。
- 实体可放 `com.myreport.entity`：`SimpleReport` / `SimpleReportBlock` / `SimpleReportRun`。

### 7.5 常量建议（应用，非 DB）

```text
SIMPLE_REPORT_ENGINE_ID_BASE = 1_000_000_000
```

## 8. 数据量与保留

| 项 | 建议 |
|----|------|
| 配置行 | 少；校级/业务条线级 |
| 区块 | 每报告通常 &lt; 20 |
| Run | 随调用增长；`plan_md` + 快照占空间；可按月归档软策略后置 |
| 文件 | 路径在库；二进制在磁盘；删 Run 是否删文件 **默认不删**（审计），运维另清 |

## 10. 风险

| 风险 | 可能性 | 影响 | 缓解 |
|------|--------|------|------|
| `engine_report_id` 与外部自传 `reportId` 撞号 | 低 | Redis 串扰 | 固定 BASE 高位段；文档告知外部勿用 ≥1e9 |
| 快照 JSON 过大 | 低 | MEDIUMTEXT 足够；超多块再限 | 区块数上限（如 50）应用校验 |
| JPA 未建生成列 | 中 | 名称唯一失效 | 显式 DDL |
| 定时与人工确认 | 中 | 堆积 PENDING | 列表筛选 + 策略 A 文档化 |
| SSRF | 高 | 安全 | 白名单在应用；库只存 URL |
