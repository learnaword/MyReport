MyReport 报告生成平台

面向高校就业质量报告场景：默认学校为武汉大学。导入毕业生去向数据，配置报告模版与指标，一键聚合生成 Word（.docx）。图表章节可由 AI 统计分析师自动撰写解读文案。

能做什么

  • 维护武汉大学毕业生就业去向明细（Excel 导入）
  • 可视化配置报告标题层级与统计指标
  • 按模版聚合数据并异步生成 Word 报告
  • 简化报告：多数据接口确认后出稿

管理台模块

  • 数据管理：导入与查看就业去向（默认武汉大学）
  • 模版管理：配置章节树与指标字段
  • 报告管理：绑定模版并一键生成下载
  • 简化报告：多数据接口 + 样式 → 计划确认 → 出 Word

推荐使用流程

完整模版报告（数据 → 模版 → 报告管理）：

1导入就业数据 → 2配置报告模版 → 3创建并生成报告

简化报告（多接口轻量出稿，详见下方专节）：

1配接口与样式 → 2生成计划 → 3确认出 Word

打开本页默认进入项目介绍；侧边栏可进入「简化报告」或各完整链路模块。

AI 能力介绍（全局）

两条路径都会用到 AI：完整模版报告与简化报告在生成 Word 时，图表数据可交给「图表统计分析师」；简化报告还可由外部 Agent 编排 plan/confirm。

图表统计分析师 读取图表上下文（strData），自动生成简洁、专业的解读与见解，嵌入报告正文。
模型与开关 默认模型 qwen-plus；可通过配置开启/关闭，未配置 API Key 时自动跳过，不影响报告生成。
失败兜底 调用失败或无结果时不中断生成流程，保证 Word 仍可正常落盘与下载。

简化报告：配置介绍

轻量路径：不必搭完整模版树。配置「多数据接口 + 渲染样式」，人工确认计划后拉数并复用现有引擎出 Word；可选邮件与目录交付。

要配什么

报告元信息
  • 名称：列表展示与计划标题
  • 交付目录：成功后落盘相对路径（如 simple/weekly）
  • 通知邮箱：可选；配置 QQ SMTP 后成功时附 .docx 发出
数据区块（可多个）
  • 标题：Word 中该段章节名
  • 接口：GET/POST + URL（host 须白名单）
  • 渲染样式：TABLE / BAR / PIE / LINE
  • 上游约定：返回 { labels, values }

管理台「简化报告 → 新建」中，区块可按一行一条填写：

学院就业人数 | GET | http://127.0.0.1:9091/demo-stat/by-college | BAR
去向分布 | GET | http://127.0.0.1:9091/demo-stat/destination | TABLE

使用流程

1新建并保存配置 → 2填写诉求并生成计划 → 3审阅 planMd → 4确认生成 Word → 5下载 / 邮件 / 交付目录
计划待确认 「生成计划」产出 MD（查什么、接口、渲染、交付），状态为待确认；未确认不出 Word。
确认后异步出稿 按区块拉数 → 组装 reportJsonArr → 复用 Spire 异步生成；可轮询运行详情或 Redis 进度。

AI 如何参与

编排侧（Agent / Skill） OpenClaw / Comate 等可按诉求调用 plan → 展示计划 → confirm，不依赖本地 python-docx 另起炉灶。
计划与人工闸门 系统根据配置与诉求生成 planMd；人（或 Agent 代操作）确认后才拉数出 Word，避免误跑。
图表解读文案 出稿时柱状/饼/折线等图表可由「图表统计分析师」（DashScope qwen-plus）写约 200 字解读嵌入正文。
